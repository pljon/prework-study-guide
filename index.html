HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The src element contains where the visible content is being pulled from.
  • The meta element describes the data on the document or webpage.
  • The footer element contains information at the bottom of the webpage.

CSS

  • div: division or section in the HTML document
  • declaration: a property followed by a value. {example-property: example-value;}
  • inline CSS: style is applied to a particular page element only
  • internal style sheet: found in header, will only apply to that webpage
  • external style sheet: style code found on a separate file, keeps content code away from style code
  • * selector: all visible elements will be effected
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Colors can be identified by the name or by hexidecimal

Git

  • git status: checks what branch we are currently on
  • git pull: fetches changes from another repository or local branch
  • git push: stages the uploaded changes from a local repository to a remote repository to be committed
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all modifications in the current branch for next commit
  • git commit -m: describes and commits the staged changes made in a local branch to the main branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer exectues code in a script.
  • An array is a group of related data.
  • Math.floor() and Math.random() can be used together to generate a random number in JavaScript.
  • Data element's length are determined by the order they were listed, starting from 0 and up by 1.
